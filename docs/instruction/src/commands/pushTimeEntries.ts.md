# src/commands/pushTimeEntries.ts 解説（初学者向け）

## このファイルの役割
日報 TL を解析して TimeTracker API に実際に送信するメインコマンドです。

## 処理の流れ
1. アクティブエディタ確認
2. 日付抽出
3. TL抽出・解析・送信データ生成
4. `user.json` から認証情報読込
5. APIで token 発行
6. 自分のユーザー情報取得（`id` または `userId`）
7. 進捗表示しながら1件ずつ POST
8. 成功後に ledger へ送信済み日付を記録

## 初学者ポイント
- `withProgress` を使うと、処理中の件数進行をユーザーに見せられます。
- API通信、パース、永続化（ledger）が1つのコマンドに統合されています。
- 途中で失敗した場合は `catch` でユーザーにエラー理由を表示します。
